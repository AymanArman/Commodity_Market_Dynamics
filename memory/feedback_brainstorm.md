name: Brainstorm Feedback
description: How to handle the brainstorm phase and what to avoid
type: feedback
---

Do not push structure onto the brainstorm. It is a flowing conversation that moves between topics organically — the user directs it.

**Why:** User explicitly corrected this expectation early in the session. Brainstorm is collaborative and emergent, not preset.

**How to apply:** Only add things to brainstorm.md when the user confirms or directs it. Do not add conversational asides or things that were floated and rejected (e.g. swap curves on the volatility page was discussed and then immediately removed — should not have been added in the first place without stronger confirmation).
